# 论文导读PPT生成器 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
论文导读PPT生成器 (Paper Guide Generator)

### 1.2 产品定位
一款面向科研人员和学生的AI驱动工具，能够自动解析学术论文PDF，生成结构化导读大纲，并根据用户选择的汇报场景智能生成PPT内容和演讲注释。

### 1.3 核心价值
- **节省时间**：将数小时的论文阅读和PPT制作压缩到分钟级别
- **结构化理解**：AI辅助提取论文核心贡献、方法、实验等关键信息
- **场景适配**：针对不同汇报场景（组会、课程、开题、跨领域）生成风格各异的内容
- **信息完整**：PPT内容详实，把事情讲清楚，把数据摆明白
- **精细粒度**：论文每个小节（如3.1.1、3.1.2）都独立拆分，不过度整合

## 2. 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 研究生 | 需要在组会汇报论文 | 快速理解论文，制作批判性分析PPT |
| 本科生 | 课程作业需要讲解论文 | 用通俗语言解释复杂概念 |
| 博士生 | 开题/综述需要梳理文献 | 对比方法演进，分析研究趋势 |
| 跨领域研究者 | 向非专业听众介绍研究 | 避免术语，强调研究意义 |

#### PDF 查看器要求
- **连续滚动**：所有页面一次性渲染到 canvas 上，用户直接滚动浏览，无需点击翻页按钮
- **高清渲染**：使用 `window.devicePixelRatio` 进行高分辨率 canvas 渲染，确保文字清晰
- **独立滚动**：PDF 面板有自己的滚动区域，不影响大纲编辑器的滚动

#### 大纲拆分规则（关键！）
- **精细拆分**：论文的每个小节标题都必须独立成为大纲节点
  - 例：3.1 下如果有 3.1.1 和 3.1.2，两者各自作为独立子节点
  - 例：Method 章节下的 Encoder、Decoder、Loss Function 各自独立
- **不过度整合**：不要把多个小节内容合并到一个节点
- **层级保持**：保持论文原有的层级结构（1级→2级→3级→...）
- **每个末级节点 → 至少一页PPT**：这是后续生成的核心规则

#### 视觉设计
- **PDF 面板**：左侧固定宽度420px，灰色边框分隔
- **大纲节点**：卡片式设计，hover 时显示操作按钮
- **层级缩进**：每级缩进 24px，连接线指示层级关系
- **当前编辑**：双击进入编辑态，输入框替换文本

#### 交互逻辑
| 操作 | 结果 |
|------|------|
| 双击节点标题 | 进入编辑模式，显示输入框 |

#### 与旧版差异（重要！）
- **移除了"导读文章"栏**：工作台只有 PDF原文 + PPT编辑 + 缩略图 三栏

- **支持内联编辑**：双击 PPT 上的标题或内容块可直接修改文字
- **编辑后自动更新笔记**：用户修改内容后，系统自动调用 AI 重新生成演讲注释

- **双击标题**：进入编辑模式，显示输入框，Enter 保存，Esc 取消

## 4. 功能需求详细

### 4.1 数据存储

### 4.2 AI 生成逻辑（核心！）

#### 大纲解析规则（paper-parse）

**精细拆分原则**：
1. 论文的每个章节标题必须独立成为大纲节点
2. 小节标题（如 3.1.1、3.1.2）必须各自独立，不能合并
3. 即使内容较短的小节也要保留为独立节点
4. 按论文原有编号层级构建树形结构

**示例**：
```
论文结构:
  3. Method
    3.1 Overall Architecture
      3.1.1 Encoder Design
      3.1.2 Decoder Design
    3.2 Loss Function

→ 大纲结构:
  3. 方法
    ├─ 3.1 整体架构
    │   ├─ 3.1.1 编码器设计    ← 独立节点
    │   └─ 3.1.2 解码器设计    ← 独立节点
    └─ 3.2 损失函数
```

#### PPT 生成规则（workspace-generate）

**输入**
- 用户编辑后的大纲（作为 prompt 骨架）
- 用户选择的模板
- 用户选择的内容密度
- 论文元信息（标题、作者、摘要等）

**输出**
- `slides[]`：PPT 幻灯片数组

**核心生成规则**
1. **每个末级节点 → 至少一页独立PPT**：
   - 大纲的每个没有子节点的末级节点必须对应至少一页 PPT
   - **绝对不要合并**多个末级节点到一页
   - 信息量大的节点可以拆成 2-3 页
2. **内容要充实详尽**：
   - 每个要点不能只写几个词，必须是完整说明句（至少15-30字）
   - 包含具体数据、实验结果、数值对比
   - 解释"为什么"和"怎么做"，而不只是"是什么"
   - 每页至少 5 个内容块
3. **布局必须多样化**（关键！）：
   - `title-points` 最多占 30% 的页面
   - 至少使用 5 种不同的布局类型
   - 方法部分优先用 `title-method-flow` 或 `title-quad`
   - 实验对比优先用 `title-two-column` 或 `title-results`
   - 研究背景/相关工作优先用 `title-timeline`
   - 核心创新点/关键发现用 `title-findings`
4. **总页数建议 15-30 页**，根据大纲节点数量调整
5. **每页内容块数量要求**：
   - cover：2-3 个
   - title-points：5-8 个
   - title-two-column：6-10 个（左右各3-5个）
   - title-quad：4 个（每个至少30字）
   - title-timeline：4-6 个
   - title-method-flow：4-6 个
   - title-findings：3-5 个
   - title-results：4-6 个

#### 图表占位说明
- 当内容涉及实验流程、模型架构、数据流等适合用图表展示的内容时，添加占位块：
  `[图表: 描述内容]`
  例如：`[图表: Transformer 编码器-解码器架构示意图]`
- 后续迭代将对接多模态 AI 模型（如 Gemini）自动生成图像替换占位符

#### 单页 AI 编辑规则（slide-ai）

**edit 模式**：
- 用户通过 prompt 指令修改当前页
- 可修改标题、布局、内容块
- 修改布局时同时调整 contentBlocks 的 type
- 同时返回更新后的 notes

**refresh-notes 模式**：
- 用户手动内联编辑 PPT 内容后自动触发
- 根据页面最新的标题和内容块重新生成演讲注释
- 返回：mainTalk、extraExplanation、transitionSentence

## 7. Edge Functions

### 7.1 paper-parse
- **功能**：解析论文文本，提取元信息和大纲结构
- **关键规则**：精细拆分每个小节标题为独立节点（见 4.2）
- **模型**：DashScope qwen3-max

### 7.2 workspace-generate
- **功能**：根据大纲 + 模板 + 密度生成完整 PPT 内容
- **关键规则**：每个末级节点至少一页PPT，布局多样化，内容充实（见 4.2）
- **模型**：DashScope qwen3-max

### 7.3 slide-ai
- **功能**：单页 PPT 的 AI 编辑 + 演讲笔记刷新
- **两种模式**：
  - `edit`：用户 prompt 指令修改页面
  - `refresh-notes`：内联编辑后自动刷新演讲注释
- **模型**：DashScope qwen3-max

### 9.2 关键依赖
- pdfjs-dist: PDF 文本提取 + canvas 高清渲染
- framer-motion: 动画效果
- sonner: Toast 通知
- lucide-react: 图标库

### 9.3 关键设计决策记录
1. **PDF 使用 canvas 渲染而非 iframe**：因为 Lovable 预览环境的沙箱限制，浏览器内置 PDF 渲染器不可用，改用 pdfjs-dist 的 canvas 渲染
2. **连续滚动而非翻页**：用户体验更接近原生 PDF 阅读器，减少交互步骤
3. **高清渲染使用 devicePixelRatio**：canvas 实际尺寸 = 显示尺寸 × devicePixelRatio，确保 Retina 屏幕文字清晰
4. **移除导读文章栏**：工作台三栏布局（PDF + PPT + 缩略图）比四栏更简洁，论文原文本身就是最好的参考
5. **内联编辑 + 自动笔记更新**：减少用户操作步骤，编辑内容后无需手动触发笔记重生成
